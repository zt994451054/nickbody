## 回滚方案

### 回滚触发条件

满足以下任一条件时立即启动回滚：

- 发布后验证中出现 P0 功能失败
- 错误率超过 <!-- 阈值，如 1% --> 且持续 5 分钟
- 数据库连接数异常或服务无法启动
- 发现数据损坏风险

### 回滚步骤

> 按反序回滚，每步确认后再执行下一步

**Step R1：服务回滚**
```bash
# 回滚至上一版本镜像
# kubectl rollout undo deployment/api-server -n production
```

**Step R2：配置回滚**
<!-- 将 Step 2 中新增的配置项恢复为原值或删除 -->

**Step R3：数据库回滚**

> ⚠️ 数据库回滚须谨慎：确认无新版本数据写入后再执行

<!-- 将 db-design.md 中的回滚 DDL 复制到此处

```sql
-- 来自 db-design.md
-- DROP TABLE IF EXISTS ...
-- ALTER TABLE ... DROP COLUMN ...
```

-->

**回滚验证**：
- [ ] 服务已恢复至上一版本，健康检查通过
- [ ] 数据库表结构已恢复
- [ ] 核心业务功能恢复正常
