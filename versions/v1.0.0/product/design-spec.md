## 全局交互规范

| 场景 | 交互行为 |
|------|---------|
| 应用形态 | menu bar 常驻（无 Dock 图标），宠物悬浮窗置顶但不抢焦点 |
| 打断原则 | 一切打断性交互由宠物情绪表达承载；**禁止系统弹窗式打断、绝不锁屏、绝不强制** |
| 摄像头激活 | 仅在跟练会话中激活，界面常驻隐私声明「🔒 On-device only」，会话结束立即释放并有明确视觉反馈 |
| 摄像头画面 | 任何界面**永不渲染真实摄像头画面**，只渲染姿态驱动的宠物空壳"影子形象" |
| 空状态 | 统计页无数据时展示宠物引导文案（"陪我做第一次运动吧"方向），不展示空表格 |
| 操作成功反馈 | 宠物动画反馈优先（开心/庆祝），辅以轻量文字；不用系统 Toast |
| 错误反馈 | 宠物困惑表情 + 简短说明 + 单一恢复动作按钮 |
| 设置变更 | 即时生效，无保存按钮 |
| 退出/隐藏 | 菜单栏可隐藏宠物或退出；跟练中可随时 Esc 退出（已完成动作计分）|

## 桌面宠物

> 对应需求见 [requirements.md #桌面宠物](./requirements.md#桌面宠物)

### 用户流程

1. 应用启动后宠物出现在上次记忆位置（默认屏幕右下）
2. 用户拖动宠物 → 松手落位并记忆；拖到屏幕边缘 → 半隐藏（露出局部）
3. 疲劳累积 → 宠物按四阶渐进变化：清醒 → 打哈欠 → 眯眼 → 趴下
4. 闲时（随机 3–8 分钟间隔）→ 触发逗趣动作（鬼脸/小动作/时间事件反应）
5. 点击宠物 → 弹出快捷菜单（查看状态 / 立即运动 / 隐藏 / 设置）

### 补充交互规则

- 悬浮窗仅宠物本体区域可交互，其余区域点击穿透
- 全屏演示/屏幕共享检测期间宠物静止且不触发逗趣（防社死）
- 多显示器：宠物归属记忆的显示器，该显示器移除后回落主屏

## 提醒邀请

> 对应需求见 [requirements.md #提醒模块](./requirements.md#提醒模块)

### 用户流程

1. 疲劳达阈值 → 宠物移动到屏幕中央 + 耍宝动画 → 弹出邀请卡片
2. 点击 "Let's move" → 进入跟练窗口
3. 点击 "5 more minutes"（每轮≤2 次）→ 卡片收起，宠物回边缘，5 分钟后再次邀请；第 2 次延迟后按钮不再出现
4. 点击卡片外/关闭 → 视为忽略：健康分进入衰减，宠物蔫态回到边缘

### 补充交互规则

- 邀请卡片为普通置顶窗口，不模态、不遮挡输入焦点，用户可继续工作
- 邀请出现时背后工作区不加遮罩（绝不制造"被锁"感）

## 运动跟练

> 对应需求见 [requirements.md #运动跟练与评分](./requirements.md#运动跟练与评分)

### 用户流程

1. 进入跟练窗口 → 首次使用请求摄像头权限（附用途+隐私说明）
   → 拒绝/无摄像头：进入降级纯跟练模式（无影子面板，全屏教练示范）
2. 授权成功 → 双面板布局：左"教练"（宠物示范），右"影子"（用户姿态镜像）
3. 逐动作流程：宠物示范 → 用户跟做 → 实时 tips（幅度/保持提示）→ 单动作完成度评价 → 下一动作
4. 6 动作完成 → 总分汇总动画 → 健康分变动展示 → 宠物反应（开心/庆祝/晋升）→ 窗口关闭回到工作
5. 中途 Esc → 确认弹层（已完成动作计分）→ 退出

### 补充交互规则

- 用户离开画面 >10s：当前动作暂停，影子面板显示回位引导
- 摄像头被抢占：toast 式提示 + 自动切换降级模式继续
- 评分状态机：`示范中 → 等待就位 → 计时评估 → 动作小结`，任何异常回落到"等待就位"
- 跟练中提醒计时暂停
