## 概述

nick V1（MVP）：在 macOS 上交付「疲劳提醒 → 宠物邀请 → 摄像头头颈操跟练评分 → 健康分/段位养成」的完整核心循环，验证"宠物情绪价值能治愈休息提醒品类留存墓地"的核心假设，并以订阅制上架 Mac App Store。

**需求来源**：
- 用户研究见 foundation/market/user-research.md（核心洞察 1-3）
- 竞品空白见 foundation/market/competitor-analysis.md（四象限分析）
- 商业策略见 foundation/business/business-model.md（慷慨免费+订阅）

## 功能需求详情

### 疲劳追踪

**优先级**：P0
**需求状态**：已确认
**测试覆盖**：待覆盖

**需求描述**：后台统计用户连续使用设备的时长（参照驾驶疲劳机制），作为提醒触发依据。只统计时长，绝不记录任何输入内容或应用信息（反监控红线）。

**涉及角色与权限**：不限角色。

**业务规则**：
1. 三档预设：Chill 90min / Standard 60min（默认）/ Strict 30min
2. 键鼠持续无操作 5 分钟 → 暂停疲劳计时；恢复操作 → 继续累计
3. 完成一次跟练 → 疲劳计时清零重新累计
4. 系统睡眠/锁屏 → 暂停计时

**验收标准**：

正向场景：
- [ ] 默认档位下连续使用 60 分钟触发提醒 <!-- TC-待关联 -->
- [ ] 离开电脑 5 分钟后计时暂停，回来后继续 <!-- TC-待关联 -->

异常场景：
- [ ] 系统睡眠唤醒后计时状态正确恢复 <!-- TC-待关联 -->

边界场景：
- [ ] 修改档位后当前计时按新档位重新计算阈值 <!-- TC-待关联 -->

**涉及数据实体**：FatigueSession（新增）
**埋点/分析需求**：无（V1 无遥测）

### 提醒模块

**优先级**：P0
**需求状态**：已确认
**测试覆盖**：待覆盖

**需求描述**：疲劳超时后由宠物以情绪化方式邀请用户运动，替代系统弹窗。绝不锁屏、绝不强制。

**业务规则**：
1. 提醒形式：宠物移动到屏幕中央 + 邀请卡片，选项 "Let's move" / "5 more minutes"
2. 延迟每轮最多 2 次，每次 +5 分钟；用完后仅剩 "Let's move"（仍可手动关闭卡片，视为忽略）
3. 忽略提醒 → 健康分进入衰减，宠物状态变蔫
4. 免打扰：检测到全屏演示/屏幕共享时自动延迟提醒

**验收标准**：

正向场景：
- [ ] 提醒出现后点击 Let's move 进入跟练 <!-- TC-待关联 -->

异常场景：
- [ ] 全屏演示模式下不弹出提醒，退出后补弹 <!-- TC-待关联 -->

边界场景：
- [ ] 第 3 次点延迟时选项不可用 <!-- TC-待关联 -->

**涉及数据实体**：ReminderEvent（新增）

### 运动跟练与评分

**优先级**：P0
**需求状态**：已确认
**测试覆盖**：待覆盖

**需求描述**：宠物领操示范头颈肩操（6 个动作：左/右侧颈拉伸、左/右转头、低头仰头、耸肩绕肩），用户跟做；摄像头姿态识别驱动"影子形象"实时镜像用户动作（**永不显示真实画面**）；逐动作实时 tips 与完成度评价，结束汇总总分。

**业务规则**：
1. 摄像头帧仅内存处理，不落盘、不上传、不入日志（架构级隐私红线）
2. 评分三维：到位幅度 / 保持时长 / 节奏；宽容带判定（如转头 55°±10° 即满分）
3. 无摄像头/拒绝授权/摄像头被占用 → 降级纯跟练模式（跟着做不评分，得基础分）
4. 跟练中可随时退出；已完成动作计分，未完成不计
5. 单次跟练时长目标 3 分钟

**验收标准**：

正向场景：
- [ ] 完成 6 个动作后显示总分与健康分变动 <!-- TC-待关联 -->
- [ ] 动作幅度不足时出现实时 tips <!-- TC-待关联 -->

异常场景：
- [ ] 拒绝摄像头授权自动进入降级模式并完成流程 <!-- TC-待关联 -->
- [ ] 跟练中摄像头被其他应用抢占 → 提示并降级 <!-- TC-待关联 -->

边界场景：
- [ ] 用户中途离开画面 10 秒 → 暂停当前动作并提示回位 <!-- TC-待关联 -->

权限场景：
- [ ] 摄像头权限仅在首次点击 Let's move 时请求，附用途说明 <!-- TC-待关联 -->

**涉及数据实体**：WorkoutSession（新增）、MoveScore（新增）

### 养成系统

**优先级**：P0
**需求状态**：已确认
**测试覆盖**：待覆盖

**需求描述**：健康分 0–1000 与六段位体系（Bronze/Silver/Gold/Platinum/Diamond/Legend，V1 实装前 3 个的形象差异），软惩罚设计。

**业务规则**：
1. 运动得分按完成质量折算健康分加分；降级模式得固定基础分
2. 跳过/忽略提醒 → 健康分每日缓慢衰减；衰减有下限，不清零
3. **段位只升不降**（软惩罚核心原则）
4. 中断后回归首次完成跟练 → "归来加成"（加分系数提高）
5. 段位晋升触发宠物庆祝动画与形象升级

**验收标准**：

正向场景：
- [ ] 总分 90+ 的跟练使健康分按规则增加 <!-- TC-待关联 -->
- [ ] 健康分达到段位阈值触发晋升庆祝 <!-- TC-待关联 -->

边界场景：
- [ ] 连续 30 天不运动，健康分衰减到下限但段位不变 <!-- TC-待关联 -->
- [ ] 衰减后回归完成跟练获得归来加成 <!-- TC-待关联 -->

**涉及数据实体**：HealthScore（新增）、RankHistory（新增）

### 桌面宠物

**优先级**：P0
**需求状态**：已确认
**测试覆盖**：待覆盖

**需求描述**：V1 拟人化电子宠物常驻桌面透明悬浮窗，可拖动/停靠于屏幕边缘；采用 8x11 规格的高密度透明 Spritesheet（帧图集）驱动状态机动效；疲劳值通过状态机动作同步呈现（清醒工作 ➔ 犯困打哈欠）。

**业务规则**：
1. 悬浮窗置顶但不抢焦点、不拦截点击（除宠物本体 192x208 像素的真实渲染区域外，其余全透明穿透）。
2. 状态机动作集（Row 0-8）：
   - Row 0: `idle` (待机呼吸、眨眼，平稳低打扰)。
   - Row 1-2: `running-right / running-left` (鼠标拖拽时，身体向拉伸方向奔跑)。
   - Row 3: `waving` (提醒做操时挥手招募)。
   - Row 4: `jumping` (打卡成功后兴奋跳跃)。
   - Row 5: `failed` (超时不运动时，耷拉脑袋，进入气馁/萎靡状态)。
   - Row 6: `waiting` (眨眼期待跟练)。
   - Row 7: `running` (用户疯狂打字时，宠物也在后台同频专注工作)。
   - Row 8: `review` (完成任务等待审阅)。
3. 16 方向眼神/头部随动（Row 9-10）：
   - 捕捉全局鼠标向量夹角（000 到 337.5 度，顺时针 16 个梯度）。
   - 切换至对应的静态视线随动帧，使宠物在桌面产生“眼神盯着用户光标”的逼真灵动效果。
4. 可在菜单栏随时隐藏/唤出宠物。
5. 动效特权付费：高阶宠物独占炫酷做操流光与特殊逗趣（如挠痒大笑）动效。

**验收标准**：

正向场景：
- [ ] 宠物可拖动到任意屏幕位置并记忆 <!-- TC-待关联 -->
- [ ] 疲劳累积时宠物状态按四阶渐进变化 <!-- TC-待关联 -->

异常场景：
- [ ] 多显示器插拔后宠物位置正确恢复 <!-- TC-待关联 -->

边界场景：
- [ ] 宠物半隐藏于屏幕边缘时点击可唤出 <!-- TC-待关联 -->

**涉及数据实体**：PetState（新增）

### Onboarding

**优先级**：P0
**需求状态**：已确认
**测试覆盖**：待覆盖

**需求描述**：首次启动流程：欢迎 → 认识宠物并命名 → 选疲劳档位（默认 Standard）→ 演示一次跟练（可跳过摄像头授权）→ 宠物入驻桌面。目标：首日完成 1 次完整跟练（激活北极星指标）。

**业务规则**：
1. 全程可跳过，跳过者使用全部默认值
2. 摄像头授权引导放在演示跟练环节，附隐私说明

**验收标准**：
- [ ] 新用户 3 分钟内完成全部引导并见到桌面宠物 <!-- TC-待关联 -->
- [ ] 跳过引导后产品处于可用默认状态 <!-- TC-待关联 -->

**涉及数据实体**：UserProfile（新增）

### 订阅

**优先级**：P1
**需求状态**：已变更
**测试覆盖**：待覆盖

**需求描述**：nick+ 订阅（$6.99/月、$39.99/年），StoreKit 2 实现，无自建服务器。免费档包含完整核心循环 + 1 只宠物 + 前 3 段位。

**业务规则**：
1. V1 付费权益：解锁高阶独占宠物及其特有的搞怪、流光等丰富动效包（作为核心付费点）、高段位（Platinum+）专属特效预告、扩展运动操位（内容随版本补充）、数据周报
2. 免费用户必须能将宠物养到 Gold（慷慨免费原则）
3. 参与 App Store Small Business Program
4. 恢复购买、家庭共享按 Apple 规范支持

**验收标准**：
- [ ] 订阅/退订/恢复购买流程在沙盒环境全部通过 <!-- TC-待关联 -->
- [ ] 免费用户核心循环无任何功能墙 <!-- TC-待关联 -->

**涉及数据实体**：无（订阅状态由 StoreKit 管理）

## 非功能需求

| 类型 | 要求 | 说明 |
|------|------|------|
| 性能 | 跟练姿态推理 ≥15fps；后台常驻 CPU <1% | 基线机型 MacBook Air M1 |
| 兼容性 | 最低 macOS 14（随 M1 验证定档）| Vision 3D 姿态 API 需求 |
| 隐私 | 摄像头帧仅内存处理，不落盘/不上传/不入日志 | 产品第一卖点，架构级保证 |
| 数据安全 | 全部数据本地 SwiftData，无云端 | GDPR/CCPA 天然合规 |
| 合规要求 | 健康声明仅"缓解疲劳/促进活动"，禁止治疗宣称 | App Store 审核红线 |
| 无障碍 | 评分宽容带覆盖活动受限用户；提供替代动作说明 | V1 最低要求 |

## 边界说明（Out of Scope）

- 不包含多宠物/形象分类货架（v2）

- 不包含账号系统与云同步

## 模块间依赖

- 提醒模块依赖疲劳追踪（触发信号）与桌面宠物（表现载体）
- 运动跟练依赖姿态识别引擎（PoseProvider）
- 养成系统依赖评分引擎输出
- 桌面宠物依赖养成系统（段位形象）与疲劳追踪（犯困可视化）
- 建议开发顺序：姿态识别 demo → 跟练评分 → 疲劳/提醒 → 养成 → 宠物 UI 精修 → 订阅
