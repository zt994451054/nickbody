## 说明

- **新增用例**：同步更新 ①「用例总览」表新增一行；②在「用例详情」区添加对应章节；两处类型、优先级保持一致
- **执行更新**（AI）：同步更新 ①用例详情执行记录；②「用例总览」表执行状态；③「用例统计」表计数
- **验收更新**（人工）：同步更新 ①用例详情验收结果；②「用例总览」表验收状态
- **执行失败附件**：失败截图/录屏按 SOP-10 上传 OSS（路径 `versions/{ver}/testing/`，命名 `YYYYMMDD-TC{序号}-描述.ext`，见 OSS.md），CDN URL 填入「工件 URL」字段
- **需求变更**（SOP-04）后：关联的 ✅ 通过 用例须退回 ♻️ 需回归，因为需求变了原测试结果可能失效，不退回等于伪装通过
- 变更追踪 → [CHANGES.md](../CHANGES.md)
