# 测试规范

> 本规范优先于公司级规范，未覆盖条目以公司级规范为准。

## 测试分层

> 单机 macOS 客户端产品，分层按桌面应用调整：

| 层级 | 说明 | 覆盖率目标 |
|------|------|----------|
| 单元测试 | 评分引擎/养成系统/疲劳计时等纯逻辑（XCTest）| 核心逻辑 ≥ 80% |
| 集成测试 | PoseProvider→评分→养成 数据链路（录制姿态数据回放驱动，不依赖真实摄像头）| 核心链路 100% |
| UI/E2E 测试 | 关键用户流程（XCUITest + 手工核心路径）| 主流程 100%（手工清单）|
| 性能测试 | 推理帧率 ≥15fps、后台 CPU <1%、内存无泄漏 | 每次发布前基线机型实测 |
| 真机场景测试 | 不同光线/距离/衣着下评分鲁棒性 | 发布前场景清单全过 |

## 用例规范

- 测试用例统一记录在 `versions/{ver}/testing/test-cases.md` 单文件中，用例编号格式：`TC-{三位序号}`
- 每个用例必须包含：前置条件、操作步骤、预期结果、优先级
- 优先级定义：P0 阻断发布 / P1 严重影响功能 / P2 一般 / P3 轻微

## 缺陷规范

- 严重级别：P0（阻断）/ P1（严重）/ P2（一般）/ P3（轻微）
- 缺陷状态流转：⏳ 待修复 → 🔄 修复中 → 🔍 待验收 → ✅ 验收通过 / ❌ 验收不通过
