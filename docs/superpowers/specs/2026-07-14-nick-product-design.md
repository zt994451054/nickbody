## 1. 产品定位

> **nick** — 住在你桌面上的健康小宠物。它记录你的屏幕疲劳，在你该动的时候带你做一套 3 分钟的头颈操，用摄像头给你的动作打分。把它养好，就是把自己的脖子养好。

- 品类：macOS 桌面健康应用（menu bar app + 桌面悬浮宠物）
- 目标用户：长时间面对电脑的办公/创作人群，居家办公者为核心画像
- 市场策略：海外优先，英文为主（中文为第二语言跟进），全球 Mac App Store 上架
- 英文卖点：*Your desk pet that keeps your neck alive.*
- 立场红线：只宣称「缓解疲劳、促进活动」，绝不宣称治疗任何疾病（医疗合规）；只统计设备使用时长，绝不做工作监控观感的功能

### 竞品定位依据（调研结论摘要）

产品横跨四个已验证品类的空白交集，无直接竞品：

| 品类 | 代表 | 验证了什么 | 缺什么 |
|---|---|---|---|
| 休息提醒 | Stretchly / Workrave / Time Out | 需求存在 | 无情绪价值，留存差 |
| 摄像头姿态监测 | SitApp / Postura / SitCoach | 端侧 AI 可行，「隐私=卖点」 | 只监测不运动、无养成 |
| 办公微运动 | Wakeout（$69.99/年，Apple 年度应用） | 付费意愿 | 无摄像头评分反馈 |
| 宠物养成 | Finch（$30M ARR，无 VC） | 养成系+订阅商业模式 | 不管你的身体 |

科学背书（可用于营销）：办公人群年颈痛发生率 45%–56%；RCT 显示颈部运动游戏干预将 6 个月颈痛发生率从 44% 降至 20%。

## 2. 核心循环

```
工作 → 疲劳累积（宠物渐进犯困）→ 提醒（宠物耍宝邀请）
  → 同意 → 摄像头跟练头颈操 → 逐动作评分 → 总分
  → 健康分变动 → 宠物状态与段位成长 → 回到工作
```

- 拒绝/忽略提醒 → 健康分缓慢衰减、宠物变蔫（打哈欠 → 趴下 → 蔫掉）
- **软惩罚原则（已确认）**：分数会衰减、宠物状态会变差，但**段位永不下降**；完成下次运动获得「归来加成」快速回血。绝不制造「欠债感」

## 3. 宠物系统

### V1 范围
- **1 只宠物**：原创物种（团状萌系小生物，介于猫与史莱姆之间；最终形象在设计稿阶段确定）
- 渲染：Rive 骨骼动画 + 状态机
- **健康分**：0–1000。运动按完成质量折算加分；跳过/超时每日缓慢衰减，衰减有下限（不清零、不清段位）
- **段位**：Bronze → Silver → Gold → Platinum → Diamond → Legend。V1 实装前 3 个段位的形象差异（配件 + 色彩 + 特效层递进），后 3 个段位后续版本解锁
- **闲时行为**：常驻屏幕边缘（可拖动、可半隐藏），随机做鬼脸、小动作，对时间事件反应（周一早上装死、周五下午跳舞）
- **疲劳可视化**：宠物的犯困程度 = 疲劳值的「预告片」

### V2+ 形象分类体系（规划，不在 V1）
- 分类货架：动物 / 植物精灵 / 原创物种
- 免费档每类开放 1–2 只基础形象；订阅解锁全部
- 付费形象差异化 = 更多表情 + 更多闲时动作库 + 段位专属特效（付费买的是「更高情绪价值密度」）
- **IP 红线**：全部原创设计，可做「致敬风格」（像素勇士风 / JRPG 风 / 赛博风的原创角色），绝不使用现有 IP 形象；IP 联名留作成熟期商务合作
- 成本控制：按体型归类共用骨骼（四足 / 双足 / 团状漂浮），同骨骼动作互相复用，新形象以换皮为主

## 4. 疲劳提醒机制

- 三档预设：Chill 90min / **Standard 60min（默认）** / Strict 30min
- 键鼠活动检测：持续无操作 5 分钟即暂停疲劳计时（人不在，不算疲劳）
- 提醒到达：宠物移动到屏幕中央耍宝邀请，选项 **"Let's move"**（开练）/ **"5 more minutes"**（延迟，每轮最多 2 次）
- **绝不锁屏、绝不强制**——紧迫感由宠物的情绪表达实现
- 演进方向（V2+）：智能疲劳值（时段权重、连续时长、上次运动质量），V1 仅做视觉呈现层

## 5. 运动系统

### 动作库（V1：1 套头颈肩操，6 个动作）
| 动作 | 主要判定信号 |
|---|---|
| 左侧颈拉伸 / 右侧颈拉伸 | 头部 roll 角 |
| 左转头 / 右转头 | 头部 yaw 角 |
| 低头仰头（chin tuck） | 头部 pitch 角 |
| 耸肩绕肩 | 肩部关键点位移轨迹 |

动作选自办公族颈痛干预研究的通用循证动作。

### 跟练界面
- 左侧：宠物领操示范（Rive 动画）
- 右侧：**影子形象**——不显示摄像头真实画面，只显示由用户姿态驱动的宠物空壳轮廓，实时镜像用户动作
- 流程：宠物示范一个动作 → 等待用户跟做 → 实时小 tips（幅度/保持提示）→ 单动作完成度评价 → 全部完成后汇总总分 → 折算健康分变动

### 评分
- 每个动作三项：到位幅度 / 保持时长 / 节奏
- **宽容带设计**：判定用区间（如转头 55°±10° 即满分），对模型差异和个体差异天然鲁棒；宽容度同时服务无障碍用户
- 降级方案：无摄像头（合盖接外屏）或拒绝授权 → 纯跟练模式（不评分，得基础分）。核心循环永不中断

## 6. 技术架构

| 层 | 选型 |
|---|---|
| 应用框架 | Swift + SwiftUI；menu bar app（LSUIElement）+ 透明无边框悬浮窗（NSPanel）承载宠物 |
| 姿态识别 | **Apple Vision framework（主选，已确认）**：FaceObservation 的 yaw/pitch/roll 直接对应转头/点头/歪头；VNDetectHumanBodyPoseRequest 肩部关键点。100% 端侧，自动调度神经引擎 |
| 识别抽象 | `PoseProvider` 协议：评分引擎只消费「角度 + 关键点」，不感知底层模型 |
| 备选方案 | M1 实测若 Vision 肩部关键点在坐姿近距画面抖动明显 → RTMPose/MoveNet 转 CoreML 打包（Apache 2.0，保住神经引擎与版本自控）。V2 跨平台用 MediaPipe（评分引擎复用） |
| 宠物渲染 | Rive macOS runtime（状态机驱动） |
| 数据 | SwiftData 本地存储；V1 无账号、无后端 |
| 付费 | StoreKit 2 订阅，无自建服务器 |
| 隐私架构 | 摄像头帧仅内存处理，不落盘、不上传；作为卖点写进 App Store 页面与官网首屏 |

明确排除的选型：YOLO 系（AGPL-3.0 闭源商用毒药）、OpenPose（非商用许可）、Electron/跨平台框架（悬浮窗/功耗短板）。

## 7. 商业模式

- **免费档（慷慨）**：完整核心循环（提醒 → 跟练 → 评分 → 养成）、1 只宠物、前 3 段位可达（免费用户能养到 Gold）
- **订阅 nick+**：$6.99/月 或 $39.99/年：全部宠物形象分类、高段位专属特效、扩展运动操库、数据周报
- 上线即带订阅；加入 App Store Small Business Program（15% 佣金）
- 订阅的价值支撑 = 持续内容供给（新形象/皮肤/运动操/季节活动），而非算力成本（端侧推理零云成本）

## 8. V1 范围与里程碑

**V1 明确砍掉**：多宠物货架、形象分类、智能疲劳值、Windows/移动端、社交/排行榜、账号与云同步。

里程碑（顺序承诺，工期不承诺）：
1. **M1 技术验证**：Vision 头部姿态跟踪 demo + Rive 悬浮窗宠物跑通；含 Vision 肩部关键点 bake-off
2. **M2 核心循环**：疲劳计时 + 提醒 + 跟练评分闭环（临时美术）
3. **M3 养成层**：健康分/段位/衰减/宠物状态机 + 正式美术接入
4. **M4 商业化 + 上架**：StoreKit、onboarding、App Store 送审
5. **M5 打磨**：TestFlight 公测反馈迭代

## 9. 资源与约束

- 独立开发，全程 AI 辅助编码
- 无美术资源：AI 生成 + Rive 社区资源 + 关键动画外包
- 设计稿：Pencil（.pen 文件入库）做静态 UI 基准；美术方向确定后用 HTML 原型验证动效，供 Rive 制作参考

## 10. 主要风险

1. **内容成本**是最大成本项与真正护城河（美术/动画矩阵）
2. 休息提醒品类历史留存差——解药是宠物必须「可爱到让人在乎」，V1 美术质量是生死线
3. 开放办公室摄像头尴尬——营销聚焦居家办公场景
4. Vision 模型版本随 macOS 走、不可锁定——宽容带评分 + 最低支持版本基线测试兜底
